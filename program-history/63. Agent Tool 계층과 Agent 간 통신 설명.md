# 63. Agent Tool 계층과 Agent 간 통신 설명

- 상태: 완료
- 작성일: 2026-06-30

## 질의

Agent/tool layer를 추가하면 Agent to Agent 통신을 하는 구조인지, 전체 구조가 어떻게 이해되어야 하는지 확인했다.

## 처리 내용

현재 프로젝트 기준으로 v1 agent 구조는 Agent to Agent 통신이 아니라 단일 Agent Orchestrator가 LLM과 tool executor를 호출하는 구조로 설명했다. Agent는 의사결정 루프, tool은 파일 읽기/쓰기/명령 실행 같은 능력, extension host는 로컬 실행 대리자, 백엔드는 정책과 감사 로그의 중심으로 구분했다.

Agent to Agent 통신은 reviewer agent, planner/coder 분리, 보안 검토 agent 같은 고급 구조에서 선택적으로 도입하는 것이며 현재 단계의 필수 요소가 아니라고 정리했다.

## 결과

Agent/tool layer 추가는 기본적으로 단일 agent가 tool을 호출하는 구조이며, Agent to Agent 통신은 나중에 필요할 때 확장하는 별도 선택지라고 답변했다.
