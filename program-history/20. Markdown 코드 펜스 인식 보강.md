# 20. Markdown 코드 펜스 인식 보강

- 상태: 완료
- 작성일: 2026-06-23

## 질의

채팅 응답에서 일반 글과 코드가 구분되지 않고 Markdown 코드 펜스가 그대로 보이는 문제를 수정해 달라는 요청이었다. 사용자는 Codex 출력 예시처럼 코드 부분과 글 부분이 분리되어 보이길 원했다.

## 처리 내용

`MarkdownMessage.tsx`, `MessageBubble.tsx`, `global.css`, `package.json`을 확인했다. `react-markdown`은 현재 의존성에 없고, 요구 범위가 fenced code block 분리 보강이라 기존 `MarkdownMessage` 파서를 유지했다. 코드 펜스가 줄 맨 앞에 있을 때만 인식되던 로직을 수정해 앞 공백이 붙은 백틱/틸드 펜스도 코드 블록으로 처리하도록 했다. 코드 블록 스타일은 너비와 헤더 영역을 정리해 일반 문단과 더 명확히 구분되도록 보강했다.

## 기능 추가/변경 결정 기록

- 요청 배경: 모델 응답의 ` ```html `, ` ```javascript ` 같은 코드 펜스가 문단 텍스트로 표시되어 코드와 설명이 구분되지 않았다.
- 선택한 기술/방식: 기존 React 컴포넌트 내부 Markdown 파서를 유지하고, 공백이 포함된 fenced code block을 인식하는 helper 함수를 추가했다.
- 검토한 대안: `react-markdown` 도입을 검토할 수 있었지만 새 의존성 설치가 필요하고 현재 수정 범위는 코드 펜스 인식 문제에 집중되어 있어 선택하지 않았다.
- 선택 이유: 기존 구조와 스타일을 유지하면서 변경 범위를 작게 제한할 수 있고, 스트리밍 중 닫히지 않은 코드 펜스 처리도 기존 동작을 유지할 수 있다.
