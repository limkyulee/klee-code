# 66. 시스템 아키텍처 이미지 갱신

- 상태: 완료
- 작성일: 2026-06-30

## 질의

변경된 설계 구조안을 바탕으로 system architecture 이미지를 생생하게 생성해 달라고 요청했다.

## 처리 내용

`docs/DESIGN.md`와 `docs/STRUCTURE_V2.md`를 확인해 새 구조의 핵심 변경점을 정리했다. Agent Orchestrator, Tool Policy, Permission/approval mode, Extension Host의 local tool executor, 중앙 LLM Gateway, MongoDB 감사/대화 저장 흐름을 반영해 새 SVG 다이어그램을 작성했다. SVG를 PNG로 렌더링해 V2 이미지로 저장하고, 기존 대표 이미지 `docs/chat-system-architecture.png`도 동일한 최신 이미지로 갱신했다.

## 기능 추가/변경 결정 기록

- 요청 배경: 기존 아키텍처 이미지가 변경된 Agent/Tool 구조와 로컬 tool 실행 경계를 반영하지 못했다.
- 선택한 기술/방식: 텍스트 정확도가 필요한 아키텍처 다이어그램이므로 생성형 이미지 대신 직접 작성한 SVG를 원본으로 두고, `sharp` 렌더링으로 PNG를 생성했다.
- 검토한 대안: `imagegen` 기반 래스터 생성을 검토했지만, 다이어그램의 한국어 텍스트와 연결선 정확도가 낮아질 수 있어 선택하지 않았다. macOS `sips` 변환도 시도했지만 SVG 이미지 추출 실패로 사용하지 않았다.
- 선택 이유: SVG 원본을 남기면 이후 구조 변경 시 수정이 쉽고, PNG는 README나 문서에 바로 사용할 수 있어 유지보수성과 사용성을 함께 확보할 수 있다.
