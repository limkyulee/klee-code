# 71. Tool calling 구현 방식 설명

- 상태: 완료
- 작성일: 2026-07-01

## 질의

Tool calling을 원래 현재처럼 구현하는지, tool이 하나하나 하드코딩되는지 질문했다.

## 처리 내용

현재 구현은 프롬프트에 tool 설명을 붙이고 `<klee_tool_call>` 태그를 파싱하는 단순 v1 방식이라고 설명했다. 일반적인 제품 수준 구현에서는 모델 API의 native tool/function calling을 사용하고, tool 정의는 registry/schema/policy/executor 구조로 관리해 코드 곳곳에 하드코딩하지 않는다고 정리했다.

현재 Klee Code 구현은 `read_file`, `search_files`가 `AgentService`, `ToolPolicyService`, `localTools.ts`에 직접 들어가 있어 하드코딩에 가까우며, 후속 개선 시 `ToolRegistry`와 공통 tool schema를 두는 방향이 맞다고 설명했다.
