# 75. 대화 계층 아키텍처 정리

- 상태: 완료
- 작성일: 2026-07-01

## 질의

현재 프로젝트 코드를 기준으로 대화 계층 아키텍처를 그려 달라는 요청을 처리했다.

## 처리 내용

확장 Webview UI, Extension Host, 백엔드 chat/agent/conversation/prompt/llm/tool/audit 계층의 주요 파일을 확인했다. 현재 채팅 입력은 Webview에서 Extension Host로 전달되고, Extension Host가 코드 컨텍스트와 `.klee` 프롬프트 컨텍스트를 구성한 뒤 `/agent/stream` SSE API를 호출하는 구조로 정리했다. 백엔드 Agent 계층은 프롬프트 조립, LLM 호출, 도구 호출 요청, 도구 결과 대기, 감사 로그와 대화 메타데이터 저장을 담당하는 것으로 설명했다.

## 변경 파일

- `program-history/75. 대화 계층 아키텍처 정리.md`

## 검증

테스트나 빌드는 실행하지 않았다. 코드 변경 없이 현재 소스 구조를 읽고 아키텍처를 문서화했다.

## 결과

사용자에게 현재 코드 기준 대화 계층 아키텍처 다이어그램과 주요 흐름을 요약해 전달했다.
