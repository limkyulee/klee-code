# 55. models 호출 시 Netty 사용 이유 설명

- 상태: 완료
- 작성일: 2026-06-29

## 질의

`/models` 호출 시 `ModelController -> LLMGateway.availableModels() -> ollamaApi.listModels()` 경로에서 왜 Netty가 사용되는지, Netty 내부 원리를 잘 모르는 상태에서 이해할 수 있게 설명해 달라는 요청이었다.

## 처리 내용

기존 확인 내용을 바탕으로 `/models` API가 로컬 메서드 호출처럼 보이지만 실제로는 Ollama 서버의 모델 목록 API로 HTTP 요청을 보내는 흐름임을 설명했다. Spring AI의 `OllamaApi`가 HTTP 클라이언트 계층을 사용하고, 그 하위 런타임에서 Reactor Netty가 소켓 연결, DNS 조회, 요청/응답 처리 등을 맡는다고 정리했다.

## 변경 파일

- `program-history/55. models 호출 시 Netty 사용 이유 설명.md`

## 검증

설명 요청이므로 별도 테스트는 실행하지 않았다.

## 결과

Netty는 `/models` 비즈니스 로직 자체가 아니라 Ollama 서버로 나가는 HTTP 통신의 하위 네트워크 엔진으로 사용된다고 설명했다.
