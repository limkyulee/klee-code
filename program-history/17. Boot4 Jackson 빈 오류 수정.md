# 17. Boot4 Jackson 빈 오류 수정

- 상태: 완료
- 작성일: 2026-06-23

## 질의

Spring ApplicationContext 시작 시 `ChatController` 생성자에서 `com.fasterxml.jackson.databind.ObjectMapper` 빈을 찾지 못해 발생하는 빌드 오류 수정을 요청했다. 이후 빌드/테스트로 재확인 후 수정해 달라고 요청했다.

## 처리 내용

`ChatController.java`, `backend/build.gradle`, 테스트 리포트, 관련 테스트 파일을 확인했다. 테스트 리포트에서 Spring Boot 4.1이 `tools.jackson.databind.json.JsonMapper` 빈(`jacksonJsonMapper`)은 자동 구성하지만, 기존 Jackson 2 타입인 `com.fasterxml.jackson.databind.ObjectMapper` 빈은 제공하지 않는 것을 확인했다. SSE 토큰 공백 보존을 위한 JSON 문자열 인코딩은 유지하되, 컨트롤러 주입 타입을 Boot 4 자동 구성과 일치하는 `JsonMapper`로 변경했다.

## 기능 추가/변경 결정 기록

- 요청 배경: 스트리밍 SSE 응답을 JSON 문자열로 인코딩하기 위해 Jackson mapper를 주입했지만, Boot 4 환경에서 `ObjectMapper` 빈이 없어 애플리케이션 컨텍스트가 시작되지 않았다.
- 선택한 기술/방식: `ChatController`에서 `tools.jackson.databind.json.JsonMapper`를 주입받고 `writeValueAsString`으로 SSE data를 JSON 문자열 인코딩한다.
- 검토한 대안: 별도 `ObjectMapper` 빈을 직접 등록하는 방식은 Boot 4의 기본 Jackson 3 자동 구성과 타입이 달라지고 추가 설정이 필요해 선택하지 않았다. `build.gradle` 의존성 변경은 이미 Jackson 3 자동 구성 빈이 존재하므로 불필요하다고 판단했다.
- 선택 이유: 기존 Boot 4 자동 구성과 일치하고, 변경 범위가 컨트롤러 한 파일로 제한되며, 기존 SSE 공백 보존 동작을 유지한다.
