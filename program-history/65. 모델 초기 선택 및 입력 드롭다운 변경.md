# 1. 모델 초기 선택 및 입력 드롭다운 변경

- 상태: 완료
- 작성일: 2026-06-30

## 질의

초기 모델을 고정값이 아니라 불러온 모델 목록의 첫 번째 항목으로 설정하고, chat input의 모델 표시를 단순 view가 아닌 드롭다운으로 교체해 달라는 요청.

## 처리 내용

`webview-ui/src/features/chat/ChatView.tsx`와 `webview-ui/src/features/chat/components/ChatInput.tsx`, `webview-ui/src/styles/global.css`를 확인했다.
모델 목록과 사용자 선호 설정을 조합해 현재 선택 모델을 계산하도록 변경했고, 선호 모델이 비어 있을 때 모델 목록의 첫 번째 항목을 `SAVE_PREFERENCES`로 저장하도록 했다.
chat input 오른쪽의 `model-pill` 표시 UI를 실제 `select` 드롭다운으로 교체해 입력창에서 모델을 바로 변경할 수 있게 했다.
설정 팝오버의 모델 기본값도 `default` 플래그 우선이 아니라 목록의 첫 번째 모델 기준으로 맞췄다.

## 기능 추가/변경 결정 기록

- 요청 배경: 초기 모델이 고정 모델처럼 동작하고, 입력창의 모델 UI가 표시 전용이라 사용자가 바로 선택할 수 없었다.
- 선택한 기술/방식: 기존 React 상태와 VS Code webview 메시지 프로토콜의 `SAVE_PREFERENCES`를 그대로 사용해 모델 선택 변경을 저장했다. 드롭다운은 네이티브 `select`로 구현했다.
- 검토한 대안: 별도 커스텀 팝오버를 만드는 방법도 가능했지만, 이미 설정 폼에서 `select`를 쓰고 있어 입력창도 같은 방식이 더 작고 안정적이라고 판단했다.
- 선택 이유: 기존 메시지 구조와 선호 설정 API를 재사용해 변경 범위를 줄였고, 접근성 및 키보드 조작이 기본 제공되는 네이티브 컨트롤로 유지보수성을 높였다.
