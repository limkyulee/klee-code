# 48. 진단 경고 정리

- 상태: 완료
- 작성일: 2026-06-29

## 질의

IDE에서 표시된 Java 불필요한 경고 억제 어노테이션과 VS Code 확장 manifest의 불필요한 activation event 경고를 수정해 달라는 요청.

## 처리 내용

`TokenService`의 `roles` 메서드가 `List<?>` 패턴 매칭과 `String::valueOf` 변환을 사용해 unchecked cast가 없음을 확인하고, 불필요한 `@SuppressWarnings("unchecked")`를 제거했다.

`extension/package.json`에서 `contributes.views`와 `contributes.commands` 선언으로 VS Code가 자동 생성하는 `activationEvents` 항목을 제거했다.

## 기능 추가/변경 결정 기록

- 요청 배경: IDE 진단에서 불필요한 경고 억제와 자동 생성 가능한 activation event 선언을 경고로 표시했다.
- 선택한 기술/방식: Java 어노테이션 제거와 VS Code 확장 manifest의 명시적 `activationEvents` 제거만 적용했다.
- 검토한 대안: 경고를 유지하거나 별도 설정으로 무시하는 방법은 실제 중복 선언을 남기므로 선택하지 않았다.
- 선택 이유: 기존 동작을 유지하면서 IDE 경고 원인만 제거하는 가장 작은 변경이며, VS Code의 contribution 기반 자동 activation 규칙과 일치한다.
