# 37. ERD 테이블 형식 재작성

- 상태: 완료
- 작성일: 2026-06-25

## 질의

사용자는 MongoDB ERD를 박스형 설명이 아니라 `collection 명`, `schema 변수 명`, `설명` 컬럼을 가진 테이블 형식으로 다시 그려 달라고 요청했다.

## 처리 내용

`mongoDB collection ERD.drawio`의 각 컬렉션 노드 라벨을 HTML 테이블 형식으로 재작성했다.
모든 컬렉션에 동일한 컬럼 구조를 적용했고, 각 스키마 행마다 컬렉션명, 변수명, 설명을 표시했다.
기존 관계선은 유지해 `users`, `conversations`, `audit_logs`, `ai_chat_memory` 간 연결을 계속 확인할 수 있게 했다.

## 기능 추가/변경 결정 기록

- 요청 배경: 컬렉션별 저장 책임과 스키마 설명을 더 명확히 보기 위해 ERD를 테이블 형식으로 바꿀 필요가 있었다.
- 선택한 기술/방식: draw.io XML의 `mxCell value`를 HTML table 라벨로 구성하고, `collection 명`, `schema 변수 명`, `설명` 컬럼을 모든 컬렉션에 반복 적용했다.
- 검토한 대안: 별도 Markdown 표 문서를 만드는 방식도 가능하지만, 사용자가 draw.io ERD 문서를 요청했으므로 기존 draw.io 파일 안에서 표현 방식을 변경했다.
- 선택 이유: draw.io에서 관계선과 컬렉션별 스키마 설명을 한 화면에서 함께 확인할 수 있고, 기존 ERD 문서 위치를 유지할 수 있다.

## 변경 파일

- `mongoDB collection ERD.drawio`
- `program-history/37. ERD 테이블 형식 재작성.md`

## 결과

MongoDB ERD가 `collection 명`, `schema 변수 명`, `설명` 컬럼을 가진 테이블 형식으로 재작성되었다.
