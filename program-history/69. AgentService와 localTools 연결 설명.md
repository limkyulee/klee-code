# 69. AgentService와 localTools 연결 설명

- 상태: 완료
- 작성일: 2026-07-01

## 질의

`AgentService`가 모델에게 `read_file` tool을 어떻게 알려주는지, `Extension Host localTools`가 실제로 실행하는 코드는 어디 있는지 질문했다.

## 처리 내용

`AgentService.toolInstructions()`가 모델에게 사용 가능한 tool 목록과 호출 JSON 형식을 프롬프트 문자열로 붙여 보내는 구조임을 확인했다. 모델이 `<klee_tool_call>{...}</klee_tool_call>` 태그를 반환하면 `AgentService`가 파싱하고, 정책 확인 후 SSE `tool_call_requested` 이벤트로 Extension Host에 전달하는 흐름을 설명했다.

Extension Host의 실제 실행 코드는 `extension/src/extension-host/chat/localTools.ts`의 `executeLocalTool()`이며, `read_file`은 `readFileTool()`, `search_files`는 `searchFilesTool()`로 분기해 실행하는 구조임을 확인했다.

## 변경 파일

- `program-history/69. AgentService와 localTools 연결 설명.md`

## 검증

테스트는 실행하지 않았다. 코드 위치와 호출 흐름만 확인했다.

## 결과

`AgentService`는 모델이 무엇이 필요한지 미리 아는 것이 아니라 tool 사용법을 프롬프트로 알려주고, 모델이 요청한 tool call을 Extension Host가 실행하는 구조라고 설명했다.
