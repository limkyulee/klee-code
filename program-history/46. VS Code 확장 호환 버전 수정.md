# 46. VS Code 확장 호환 버전 수정

- 상태: 완료
- 작성일: 2026-06-29

## 질의

VS Code 1.118.1에서 개발 중인 확장을 로드할 때 확장이 `^1.125.0`을 요구해 호환되지 않는다는 오류를 해결해 달라는 요청.

## 처리 내용

`extension/package.json`의 `engines.vscode` 값을 확인했고, 현재 실행 중인 VS Code 1.118.1보다 높은 `^1.125.0`으로 선언되어 있어 로드가 차단되는 것을 확인했다. 확장의 최소 VS Code 버전을 `^1.118.0`으로 낮추고, `extension/package-lock.json`에 기록된 루트 패키지의 동일한 엔진 메타데이터도 함께 갱신했다.

## 기능 추가/변경 결정 기록

- 요청 배경: 개발 중인 VS Code 확장이 현재 설치된 Code 1.118.1에서 호환성 검사에 실패했다.
- 선택한 기술/방식: 확장 매니페스트의 `engines.vscode` 범위를 `^1.118.0`으로 변경하고 lockfile 루트 메타데이터를 동기화했다.
- 검토한 대안: VS Code를 1.125.0 이상으로 업데이트하는 방식도 가능하지만, 현재 개발 환경에서 즉시 로드하려는 요청이므로 확장 호환 범위를 낮추는 방식을 선택했다.
- 선택 이유: 변경 범위가 작고, VS Code 확장 로드 실패의 직접 원인인 매니페스트 호환성 선언을 수정하는 방식이라 유지보수성과 기존 구조 측면에서 가장 단순하다.

## 변경 파일

- `extension/package.json`
- `extension/package-lock.json`
- `program-history/46. VS Code 확장 호환 버전 수정.md`

## 검증

`extension` 디렉터리에서 `npm run compile`을 실행했고 타입 체크, 린트, 확장 빌드, 웹뷰 빌드가 모두 성공했다.

## 결과

VS Code 1.118.1에서 확장 개발 로드가 매니페스트 호환성 요구 버전 때문에 차단되지 않도록 수정했다.
