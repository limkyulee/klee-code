# 63. Clear 명령으로 대화 초기화

- 상태: 완료
- 작성일: 2026-06-30

## 질의

`/clear` 스킬을 추가해 대화 내역과 context window를 초기화해 달라는 요청이었다.

## 처리 내용

`/clear`는 모델에 전달되는 프롬프트 스킬이 아니라 extension host에서 처리하는 로컬 명령으로 구현했다. 채팅 입력의 첫 토큰이 `/clear`이면 현재 conversationId의 진행 중 요청을 중단하고, 로컬 스냅샷을 삭제하며, 백엔드 대화 삭제 API를 호출해 `ChatMemory.clear(conversationId)`까지 수행되도록 했다. 아직 저장되지 않은 대화라서 404가 발생하는 경우는 무시하고 새 conversationId를 생성한다.

`docs/DESIGN_V3.md`에도 `/clear`가 모델에 전달되지 않는 초기화 명령임을 문서화했다.

## 기능 추가/변경 결정 기록

- 요청 배경: 사용자가 slash skill 방식으로 대화 내역과 context window를 초기화하고 싶어 했다.
- 선택한 기술/방식: extension host에서 `/clear`를 가로채고, 기존 대화 삭제 API와 `CONVERSATION_RESET` UI 이벤트를 재사용했다.
- 검토한 대안: 백엔드 내부 프롬프트 스킬로 `clear.md`를 추가하는 방식은 실제 대화 메모리를 비우지 못하므로 선택하지 않았다.
- 선택 이유: context window는 conversationId와 Spring AI chat memory에 연결되므로, 모델 호출 전에 현재 대화를 삭제하고 새 conversationId로 전환하는 방식이 가장 직접적이다.
