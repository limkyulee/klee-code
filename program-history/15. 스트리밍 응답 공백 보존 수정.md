# 15. 스트리밍 응답 공백 보존 수정

- 상태: 완료
- 작성일: 2026-06-23

## 질의

사용자가 AI 답장이 `Hello!I'manAIassistant...`처럼 띄어쓰기 없이 노출되므로 스트리밍 응답 처리에서 `trim` 처리를 빼 달라고 요청했다.

## 처리 내용

`extension/src/extension-host/services/chatApiClient.ts`의 SSE 파싱 로직을 확인했다.
스트림의 `data:` 값을 읽을 때 전체 값을 `trimStart()`로 정리하면 토큰 앞의 실제 공백까지 제거되어 단어가 붙는 문제가 생길 수 있다고 판단했다.
SSE 필드 규칙에 맞춰 콜론 뒤 구분용 공백 한 칸만 제거하고, 모델 토큰의 선행 공백은 그대로 보존하도록 수정했다.
남은 버퍼 처리도 `buffer.trim()` 대신 `buffer.length > 0`으로 바꿔 공백만 있는 데이터가 임의로 버려지지 않게 했다.

## 기능 추가/변경 결정 기록

- 요청 배경: 스트리밍으로 받은 LLM 응답 토큰의 선행 공백이 제거되어 문장 내 단어들이 붙어 보이는 문제를 해결하기 위한 요청이었다.
- 선택한 기술/방식: 확장 호스트의 SSE 클라이언트에서 `event:`/`data:` 필드 값을 파싱할 때 구분용 공백 한 칸만 제거하는 헬퍼를 사용했다.
- 검토한 대안: 백엔드에서 토큰 사이에 공백을 재삽입하는 방식은 모델이 반환한 원문을 변경하고 언어/구두점별 예외가 많아 선택하지 않았다.
- 선택 이유: 문제는 전송 포맷 파싱 단계에서 실제 토큰 공백을 잃는 것이므로, 클라이언트 파서에서 원문 공백을 보존하는 방식이 가장 작고 기존 구조와 일관적이다.
