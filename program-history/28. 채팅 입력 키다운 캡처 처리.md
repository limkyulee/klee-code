# 28. 채팅 입력 키다운 캡처 처리

- 상태: 완료
- 작성일: 2026-06-23

## 질의

`ChatInput.tsx`의 textarea에서 `onKeyDown`이 적용되지 않는 것 같고, 하드코딩이 아닌 `@keydown.event` 같은 방식이 있는지 다시 확인해 달라는 요청.

## 처리 내용

- `extension/webview-ui/src/features/chat/components/ChatInput.tsx`와 웹뷰 UI 패키지 설정을 확인했다.
- 프로젝트가 React 19 + TSX 구조라 Vue/Svelte식 `@keydown.enter` 이벤트 modifier는 사용할 수 없음을 확인했다.
- textarea 키 입력 처리를 React에서 지원하는 캡처 단계 이벤트인 `onKeyDownCapture`로 변경했다.
- Enter 전송 조건은 `shouldSubmitFromKeyDown` 함수로 분리하고, 디버그용 `console.log`를 제거했다.
- IME 조합 중 Enter 입력은 기존처럼 전송하지 않도록 유지했다.

## 기능 추가/변경 결정 기록

- 요청 배경: textarea의 `onKeyDown` 동작이 웹뷰에서 적용되지 않는 것으로 보여 Enter 전송 처리를 더 안정적으로 잡기 위한 요청이었다.
- 선택한 기술/방식: React TSX에서 지원되는 `onKeyDownCapture`를 textarea에 연결하고, 전송 키 판단 로직을 별도 predicate로 분리했다.
- 검토한 대안: `@keydown.enter` 문법은 Vue/Svelte 템플릿 문법이라 현재 React TSX 코드베이스에 적용할 수 없어 제외했다. native `addEventListener`를 ref에 직접 붙이는 방식은 React 렌더링 흐름과 분리되어 우선 적용하지 않았다.
- 선택 이유: 기존 React 컴포넌트 구조를 유지하면서 이벤트 전파의 캡처 단계에서 먼저 처리할 수 있고, 변경 범위를 `ChatInput.tsx`에 한정할 수 있다.

## 결과

textarea 키 입력을 `onKeyDownCapture`로 처리하도록 바꾸고, React TSX에서는 `@keydown.enter` 문법을 사용할 수 없다는 점을 반영해 기존 구조 안에서 수정했다.
