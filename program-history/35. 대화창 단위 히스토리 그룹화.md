# 35. 대화창 단위 히스토리 그룹화

- 상태: 완료
- 작성일: 2026-06-24

## 질의

사용자는 한 대화창에서 여러 번 질문해도 히스토리에는 하나의 대화로 보이도록, `/audit/chat-history`를 `conversationId` 단위 요약 목록으로 재설계한 계획을 구현해 달라고 요청했다.

## 처리 내용

백엔드 audit 히스토리 반환 흐름과 extension 히스토리 표시 타입을 확인했다.
`AuditLogService.recentChatHistory()`가 기존에는 `audit_logs` 개별 문서를 그대로 반환하던 구조였으므로, 같은 `conversationId`의 로그를 하나의 대화 요약으로 묶도록 변경했다.
대화 요약은 최초 질문 시각과 제목, 마지막 갱신 시각, 마지막 턴 상태, 턴 수를 포함한다.
프론트엔드 히스토리 타입은 `updatedAt`, `turnCount`를 받도록 바꾸고, 목록 key와 시간 표시 기준을 `conversationId`, `updatedAt`으로 수정했다.

## 기능 추가/변경 결정 기록

- 요청 배경: 한 대화창에서 여러 질문이 발생하면 히스토리에 질문 수만큼 항목이 생겨 사용자 관점의 대화 단위와 맞지 않았다.
- 선택한 기술/방식: `audit_logs`는 턴 단위 원본 기록으로 유지하고, `/audit/chat-history` 응답만 `conversationId` 기준 대화 요약으로 계산한다.
- 검토한 대안: 별도 `conversations` 컬렉션을 추가하는 방식도 가능하지만, 현재 요구는 목록 중복 제거가 핵심이고 상세 복원/제목 수정/삭제 기능은 아직 없으므로 범위가 과하다고 판단했다.
- 선택 이유: 기존 감사 로그와 Spring AI ChatMemory 구조를 유지하면서 UI 히스토리 표시 단위만 바로잡을 수 있어 변경 범위가 작고 안전하다.
