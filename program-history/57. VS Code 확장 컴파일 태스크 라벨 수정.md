# 57. VS Code 확장 컴파일 태스크 라벨 수정

- 상태: 완료
- 작성일: 2026-06-30

## 질의

`Could not find the task 'extension: compile'.` 오류가 발생하지 않도록 설정을 수정해 달라는 요청.

## 처리 내용

`.vscode/launch.json`의 `preLaunchTask`가 `extension: compile`을 참조하고 있음을 확인했다.
`.vscode/tasks.json`의 실제 태스크 라벨이 `npm: compile - extension`으로 되어 있어 VS Code가 태스크를 찾지 못하는 상태였다.
디버그 설정의 의도에 맞춰 `.vscode/tasks.json`의 태스크 `label`을 `extension: compile`로 변경했다.

## 기능 추가/변경 결정 기록

- 요청 배경: VS Code 디버그 실행 시 `extension: compile` 태스크를 찾지 못하는 오류가 발생했다.
- 선택한 기술/방식: 기존 npm 태스크 정의는 유지하고, VS Code 태스크 라벨만 `preLaunchTask` 값과 동일하게 수정했다.
- 검토한 대안: `launch.json`의 `preLaunchTask`를 `npm: compile - extension`로 바꾸는 방법도 가능했지만, 현재 디버그 설정이 이미 명확한 이름을 사용하고 있어 태스크 라벨을 맞추는 방식을 선택했다.
- 선택 이유: 빌드 스크립트와 실행 흐름은 건드리지 않고 VS Code 태스크 매칭 문제만 최소 범위로 해결할 수 있다.

## 변경 파일

- `.vscode/tasks.json`
- `program-history/57. VS Code 확장 컴파일 태스크 라벨 수정.md`

## 검증

`sed -n '1,80p' .vscode/tasks.json`와 `sed -n '1,40p' .vscode/launch.json`로 `label`과 `preLaunchTask`가 모두 `extension: compile`로 일치하는지 확인했다.
별도 빌드나 테스트는 실행하지 않았다.

## 결과

VS Code가 디버그 실행 전 `extension: compile` 태스크를 정상적으로 찾을 수 있도록 설정을 수정했다.
