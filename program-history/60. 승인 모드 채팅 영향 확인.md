# 60. 승인 모드 채팅 영향 확인

- 상태: 완료
- 작성일: 2026-06-30

## 질의

`ChatInput.tsx`의 `Ask for approval` 영역이 현재 채팅 동작에 영향을 주는지 확인했다.

## 처리 내용

`extension/webview-ui/src/features/chat/components/ChatInput.tsx`에서 승인 모드 UI 상태와 전송 로직을 확인했다. `selectedModeId`는 컴포넌트 내부 표시 상태로만 사용되고, 메시지 전송 시 `onSend(trimmedText)`로 텍스트만 전달되는 것을 확인했다.

상위 컴포넌트인 `extension/webview-ui/src/features/chat/ChatView.tsx`와 프로토콜 파일 `extension/webview-ui/src/features/chat/api/webviewProtocol.ts`, 확장 호스트 핸들러 `extension/src/extension-host/webview/ChatWebviewMessageHandler.ts`를 확인해 `SEND_MESSAGE` payload가 `{ text }`만 포함하며 승인 모드 값은 전달되지 않는 것을 확인했다.

## 변경 파일

- `program-history/60. 승인 모드 채팅 영향 확인.md`

## 검증

- `rg -n "Ask for approval|approval|approv|sandbox|permission" extension/webview-ui/src -S`
- `rg -n "<ChatInput|onSend=|function .*onSend|handle.*Send|sendMessage|postMessage" extension/webview-ui/src extension/src -S`
- `rg -n "SEND_MESSAGE|WebviewToExtensionMessage|type: 'SEND_MESSAGE'|payload: \\{ text" extension -S`

## 결과

현재 `Ask for approval` 영역은 UI 표시와 선택 상태 변경에만 영향을 주며, 실제 채팅 요청이나 승인 정책에는 연결되어 있지 않다고 정리했다.
