# 11. Codex 지침을 .codex로 이동

- 상태: 완료
- 작성일: 2026-06-23

## 질의

사용자가 기존 루트 `AGENTS.md`의 Codex 지침을 `.codex/` 아래로 옮기고, `rules`로 분리할 수 있는 부분이 있으면 반영해 달라고 요청했다.

## 처리 내용

OpenAI Codex 공식 문서의 `AGENTS.md`와 `rules` 안내를 확인한 뒤, 루트 `AGENTS.md`의 내용을 `.codex/AGENTS.md`로 이동했다.
현재 문서 내용은 저장소 지침과 기록 규칙 중심이어서, command permission 제어용 `rules/*.rules`로 옮길 만한 항목은 없다고 판단했다.
작업 트리의 기존 변경 사항은 건드리지 않았고, 이번 작업과 직접 관련된 파일만 정리했다.

## 기능 추가/변경 결정 기록

- 요청 배경: Codex가 프로젝트별 지침을 `.codex/`에서 읽도록 저장소 구조를 맞추려는 요청이었다.
- 선택한 기술/방식: 프로젝트 지침을 `.codex/AGENTS.md`로 두고, command 허용/차단 규칙에 해당하는 내용이 없으므로 별도 rules 파일은 만들지 않았다.
- 검토한 대안: 루트 `AGENTS.md`를 그대로 두는 방식과, 내용 일부를 `.codex/rules/*.rules`로 억지 분리하는 방식을 검토했지만 중복과 의미 없는 규칙 파일 생성을 피했다.
- 선택 이유: 저장소 지침과 command 정책을 분리하는 Codex 문서 구조에 맞추되, 실제로 규칙으로 표현 가능한 내용만 남기는 편이 유지보수성이 높다.

## 변경 파일

- `.codex/AGENTS.md`
- `AGENTS.md`

## 검증

- `node /Users/limkyulee/.codex/skills/.system/openai-docs/scripts/fetch-codex-manual.mjs` 실행 시 네트워크 DNS 문제로 `developers.openai.com`에 접근하지 못해 실패했다.
- `sed -n '1,240p' AGENTS.md`로 기존 내용 확인 후 `.codex/AGENTS.md`로 반영했다.

## 결과

루트 `AGENTS.md`의 Codex 지침을 `.codex/AGENTS.md`로 옮겼고, 현재 내용상 별도 `rules` 파일은 만들지 않았다.
