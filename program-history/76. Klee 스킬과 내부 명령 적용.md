# 76. Klee 스킬과 내부 명령 적용

- 상태: 완료
- 작성일: 2026-07-01

## 질의

Claude Code의 tool calling 분석 방식을 참고해 사용자 커스텀 스킬/rule을 `.klee` 폴더에서 읽고, 내부 slash command 기능을 적용해 달라는 요청이었다.

## 처리 내용

`extension/src/extension-host/chat/kleeContext.ts`, `slashCommand.ts`, `ChatWebviewMessageHandler.ts`와 관련 테스트를 확인했다. 기존 `.klee/rules`, `.klee/skills`, `.klee/hooks` 구조는 유지하면서 컨텍스트 파일을 path 기준으로 정렬해 backend에 안정적으로 전달하도록 했다. 내부 slash command registry에는 `/help`, `/status`를 추가했고, 두 명령은 모델 호출 없이 extension host에서 채팅 응답으로 처리되도록 했다. Backend prompt 조립은 기존 구조를 유지하되, slash command 없이도 project rule/hook이 포함되고 custom skill은 호출된 경우에만 포함되는 동작을 테스트로 고정했다.

## 기능 추가/변경 결정 기록

- 요청 배경: Claude Code식 구조처럼 커스텀 지시 파일과 내부 명령을 명확한 registry/컨텍스트 흐름으로 적용하려는 요청이었다.
- 선택한 기술/방식: `.klee/rules/*.md`, `.klee/skills/*.md`, `.klee/hooks/*.md`를 공식 구조로 유지하고, extension-local slash command는 `LOCAL_SLASH_COMMANDS`에서 관리하도록 했다.
- 검토한 대안: Claude Code의 native `tool_use/tool_result`, 지연 로딩, 훅/권한 파이프라인까지 한 번에 도입하는 방식은 현재 XML 기반 tool loop와 범위가 크게 달라 제외했다.
- 선택 이유: 기존 구현과 호환되며, 모델 호출 전 처리해야 하는 내부 명령과 backend prompt에 합성되는 스킬/rule 책임을 분리할 수 있다.
