# 58. VS Code extension compile 태스크 원인 확인

- 상태: 완료
- 작성일: 2026-06-30

## 질의

VS Code 실행 시 `Could not find the task 'extension: compile'.` 오류가 발생하는 원인을 확인해 달라는 요청.

## 처리 내용

`.vscode/launch.json`, 루트 `.vscode/tasks.json`, `extension/.vscode/tasks.json`, `extension/package.json`을 확인했다.

루트 `.vscode/launch.json`은 `preLaunchTask`로 `extension: compile`을 참조하고, 루트 `.vscode/tasks.json`에는 같은 라벨의 태스크가 존재했다. 하지만 해당 `tasks.json`은 Git 추적 대상이 아니며 `.gitignore`의 `.vscode/*` 규칙에 의해 ignored 상태였다. 따라서 다른 체크아웃 환경이나 해당 파일이 없는 워크스페이스에서는 VS Code가 `extension: compile` 태스크를 찾지 못한다.

반면 `extension/.vscode/tasks.json`에는 `npm: compile` 라벨의 태스크가 있고, `extension/.vscode/launch.json`도 `npm: compile`을 참조한다. 즉 루트 워크스페이스 실행 설정과 extension 하위 워크스페이스 실행 설정의 태스크 라벨이 서로 다르다.

## 변경 파일

- `program-history/58. VS Code extension compile 태스크 원인 확인.md`

## 검증

다음 명령으로 설정과 Git 추적/ignore 상태를 확인했다.

- `rg -n "extension: compile|compile" .vscode extension package.json --hidden`
- `sed -n '1,200p' .vscode/launch.json`
- `sed -n '1,220p' .vscode/tasks.json`
- `sed -n '1,220p' extension/.vscode/tasks.json`
- `git ls-files .vscode/tasks.json .vscode/launch.json extension/.vscode/tasks.json extension/.vscode/launch.json`
- `git check-ignore -v .vscode/tasks.json .vscode/launch.json extension/.vscode/tasks.json extension/.vscode/launch.json`

## 결과

오류 원인은 `preLaunchTask`가 참조하는 `extension: compile` 태스크 정의가 현재 루트에는 로컬 ignored 파일로만 존재해, VS Code가 사용하는 워크스페이스 또는 다른 환경에서 로드되지 않는 구조라는 점으로 확인했다.
