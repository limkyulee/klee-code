# 38. Entity Relation Table ERD 재작성

- 상태: 완료
- 작성일: 2026-06-25

## 질의

사용자는 MongoDB ERD를 단순 HTML 표가 아니라 draw.io의 Entity Relation Table 형태로 다시 그려 달라고 요청했다.

## 처리 내용

`mongoDB collection ERD.drawio`를 Entity Relation Table 구조에 가깝게 재작성했다.
각 컬렉션은 `shape=table` 컨테이너로 구성했고, 내부 행은 `shape=tableRow`와 `shape=partialRectangle` 셀로 분리했다.
테이블 제목에는 컬렉션명을 표시하고, 각 행은 `Key`, `schema 변수 명`, `설명` 열로 구성했다.
`PK`, `FK`, `UQ`, `TTL` 같은 키 정보를 첫 번째 열에 표시했으며, 기존 컬렉션 간 관계선은 유지했다.

## 기능 추가/변경 결정 기록

- 요청 배경: ERD를 draw.io에서 편집 가능한 Entity Relation Table 형태로 보고 싶다는 요청이 있었다.
- 선택한 기술/방식: draw.io XML에서 각 컬렉션을 `shape=table` 컨테이너와 행/셀 노드로 구성했다.
- 검토한 대안: HTML table 라벨만 사용하는 방식은 간단하지만 draw.io ER 테이블처럼 행과 셀을 개별 편집하기 어렵다.
- 선택 이유: Entity Relation Table 구조에 맞춰 PK/FK와 스키마 설명을 분리해 표시할 수 있고, draw.io에서 ERD 문서로 다루기 쉽다.

## 결과

MongoDB ERD가 Entity Relation Table 형태의 테이블 컨테이너, 행, 셀 구조로 재작성되었다.
