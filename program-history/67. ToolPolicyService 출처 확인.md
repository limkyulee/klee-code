# 67. ToolPolicyService 출처 확인

- 상태: 완료
- 작성일: 2026-07-01

## 질의

`ToolPolicyService.java`가 무엇인지, 사용자가 이전에 Tool calling 구현을 요청했는지 확인했다.

## 처리 내용

`backend/src/main/java/com/kleecode/backend/tool/service/ToolPolicyService.java`와 관련 `AgentService` 구현을 확인했다. `program-history/64. Tool calling Agent 구조 적용.md`를 확인해, Ask for approval 드롭다운과 tool calling 관계를 이해한 뒤 실제 프로젝트에 tool calling 구조 적용을 요청했다는 기록이 있음을 확인했다.

`ToolPolicyService`는 v1 agent tool calling에서 허용된 로컬 tool인지 검사하고, 현재는 읽기 전용 tool인 `read_file`, `search_files`만 승인 없이 실행 가능하게 제한하는 정책 서비스다.

## 변경 파일

- `program-history/67. ToolPolicyService 출처 확인.md`

## 검증

테스트는 실행하지 않았다. 파일 내용과 히스토리 기록만 확인했다.

## 결과

히스토리 기준으로 Tool calling 구현 요청이 있었고, `ToolPolicyService`는 그 구현 중 백엔드 tool 실행 정책을 담당하는 파일이라고 답변했다.
