# 21. Markdown 코드 복사와 색상 표시

- 상태: 완료
- 작성일: 2026-06-23

## 질의

`MarkdownMessage.tsx`의 코드 블록 영역 오른쪽 상단에 복사 아이콘을 추가하고, 코드 언어별로 흰색 단색이 아닌 색상 스타일이 표시되도록 수정해 달라는 요청.

## 처리 내용

`extension/webview-ui/src/features/chat/components/MarkdownMessage.tsx`를 확인해 외부 Markdown 라이브러리 없이 직접 코드 펜스를 파싱하고 있음을 확인했다. 코드 블록 렌더링을 `CodeBlockView`로 분리하고, 헤더 오른쪽에 복사 버튼을 추가했다. `navigator.clipboard.writeText`를 우선 사용하고, 불가능한 경우 숨겨진 textarea와 `execCommand('copy')`를 fallback으로 사용하도록 했다.

코드 내용은 React 노드로 안전하게 토큰화해 `span.syntax-*` 클래스를 적용하도록 변경했다. HTML/XML, CSS, JSON, Shell, SQL, Python, Java/Kotlin, TypeScript/JavaScript 계열에서 주석, 문자열, 키워드, 타입, 함수, 숫자, 속성, 연산자 등을 구분해 색상을 적용한다.

`extension/webview-ui/src/styles/global.css`에는 코드 블록 헤더, 복사 아이콘, 복사 성공/실패 상태, 구문 색상 스타일을 추가했다.

## 기능 추가/변경 결정 기록

- 요청 배경: 코드 블록에서 사용자가 코드 내용을 쉽게 복사하고, 언어별 코드가 모두 흰색으로 표시되는 문제를 개선하기 위함.
- 선택한 기술/방식: 기존 직접 Markdown 파서 구조를 유지하고, React 상태 기반 복사 버튼과 정규식 기반 경량 구문 하이라이터를 추가했다. 색상은 VS Code Webview 테마 변수와 fallback 색상을 함께 사용했다.
- 검토한 대안: Prism, Shiki 같은 외부 syntax highlighter 의존성을 추가하는 방식을 고려할 수 있으나, 현재 확장 의존성이 작고 Markdown 렌더러도 자체 구현이라 번들 크기와 적용 범위를 줄이기 위해 선택하지 않았다.
- 선택 이유: 기존 구조와 일관되고 변경 범위가 작다. 별도 패키지 설치 없이 타입체크와 린트를 통과하며, VS Code 테마 변수 기반이라 웹뷰 테마와도 자연스럽게 맞는다.
