# 23. 채팅 입력창 모델 표시와 중단 버튼

- 상태: 완료
- 작성일: 2026-06-23

## 질의

채팅 입력창을 참고 이미지처럼 바꾸고, 현재 AI 모델 표시, 승인/추론 방식 드롭다운, 전송 중 멈춤 아이콘 전환, 멈춤 클릭 시 현재 응답 중단 기능을 요청했다.

## 처리 내용

- `ChatInput.tsx`, `ChatView.tsx`, `global.css`를 확인하고 composer 하단 액션 바를 새로 구성했다.
- 입력창 왼쪽에 새 대화용 `+` 아이콘 버튼을 배치하고, 기존 header의 `+` 버튼은 제거했다.
- `Ask for approval`, `Approve for me`, `Full access`를 선택할 수 있는 드롭다운 UI를 추가했다.
- 오른쪽에는 백엔드에서 받은 현재 모델명을 표시하고, 전송 버튼은 pending 중 정지 아이콘으로 바뀌게 했다.
- `STOP_GENERATION` 메시지를 webview-extension 프로토콜에 추가하고, extension host에서 현재 스트리밍 요청의 `AbortController`를 보관해 중단 버튼 클릭 시 fetch를 abort하도록 했다.
- 백엔드 SSE 연결 종료/오류/타임아웃 시 reactive 구독을 dispose해 모델 스트림도 중단되도록 보강했다.
- `/chat/status` 엔드포인트와 `ChatStatus` record를 추가해 현재 provider/model 설정을 webview에 전달했다.

## 기능 추가/변경 결정 기록

- 요청 배경: 사용자가 참고 이미지와 같은 입력창 구조와 응답 중단 버튼을 요청했다.
- 선택한 기술/방식: webview composer를 React 상태 기반 드롭다운과 pending 상태 기반 아이콘 버튼으로 구성하고, extension host의 `AbortController`를 `fetch` signal에 연결했다. 현재 모델 표시는 백엔드 `GET /chat/status`에서 내려주는 설정값을 사용했다.
- 검토한 대안: webview에서 UI만 멈춘 것처럼 표시하는 방식은 실제 모델 호출이 계속될 수 있어 제외했다. 모델명을 extension 설정에 하드코딩하는 방식은 백엔드의 실제 provider/model 설정과 어긋날 수 있어 제외했다.
- 선택 이유: 기존 스트리밍 구조를 유지하면서 변경 범위를 작게 가져갈 수 있고, 클라이언트 abort와 서버 구독 dispose를 함께 처리해 실제 중단 동작에 더 가깝다.
