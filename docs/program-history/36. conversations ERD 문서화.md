# 36. conversations ERD 문서화

- 상태: 완료
- 작성일: 2026-06-25

## 질의

사용자는 `conversations` 컬렉션을 추가하는 이유가 잘 이해되지 않으므로, 기존 draw.io MongoDB ERD 문서에 `conversations` 컬렉션과 저장 책임 분리를 시각적으로 반영해 달라고 요청했다.

## 처리 내용

기존 `mongoDB collection ERD.drawio`를 확인하고, 새 파일을 만들지 않고 해당 draw.io XML을 갱신했다.
`conversations` 컬렉션을 중앙에 추가해 히스토리 목록, 삭제, 제목, 대화 상태를 관리하는 메타데이터 저장소로 표시했다.
`audit_logs`는 질문/응답 본문 원본과 컨텍스트를 저장하는 턴 단위 컬렉션으로 명확히 표시했고, `ai_chat_memory`는 Spring AI가 관리하는 모델 이어가기용 메모리임을 표시했다.

## 기능 추가/변경 결정 기록

- 요청 배경: `conversations`, `audit_logs`, `ai_chat_memory`의 역할 차이를 텍스트 설명만으로 이해하기 어려워 ERD 문서가 필요했다.
- 선택한 기술/방식: 기존 draw.io XML 파일에 `conversations` 노드와 `users -> conversations`, `conversations -> audit_logs`, `conversations -> ai_chat_memory` 관계를 추가했다.
- 검토한 대안: 별도 ERD 파일을 새로 만드는 방식도 가능하지만, 기존 MongoDB 컬렉션 ERD가 이미 있어 문서가 분산되지 않도록 기존 파일을 갱신했다.
- 선택 이유: 현재 저장소의 문서 구조를 유지하면서 대화 메타데이터, 질문/응답 원본, 모델 메모리의 책임 분리를 한 화면에서 확인할 수 있다.

## 변경 파일

- `mongoDB collection ERD.drawio`
- `program-history/36. conversations ERD 문서화.md`

## 검증

- draw.io XML 구조를 유지하도록 기존 `mxfile`, `diagram`, `mxGraphModel`, `mxCell` 형식 안에서만 노드와 관계를 추가했다.
- 백엔드 구현 파일은 수정하지 않았다.

## 결과

MongoDB ERD에 `conversations` 컬렉션과 관련 관계를 추가해, 대화 메타데이터는 `conversations`, 질의/응답 본문은 `audit_logs`, 모델 이어가기 메모리는 `ai_chat_memory`에 저장된다는 구조를 문서화했다.
