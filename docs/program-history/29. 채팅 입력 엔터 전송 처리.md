# 29. 채팅 입력 엔터 전송 처리

- 상태: 완료
- 작성일: 2026-06-23

## 질의

`extension/webview-ui/src/features/chat/components/ChatInput.tsx`의 textarea에서 Enter 입력 시 메시지를 전송하고, Shift+Enter 입력 시 줄바꿈이 되도록 참고 URL의 방식에 맞춰 처리해 달라는 요청.

## 처리 내용

`ChatInput.tsx`를 확인해 textarea 키다운 처리와 form submit 흐름을 검토했다. 참고 글의 핵심 조건인 `Enter` 입력과 `shiftKey` 미사용 여부를 기준으로, 일반 Enter에서는 `preventDefault()` 후 submit을 호출하고 Shift+Enter는 기본 textarea 줄바꿈 동작에 맡기는 구조를 확인했다. 키 이벤트 연결은 textarea의 일반 `onKeyDown`에서 처리되도록 정리했다.

## 기능 추가/변경 결정 기록

- 요청 배경: textarea에서 Enter만 누르면 전송하고 Shift+Enter는 줄바꿈으로 남겨야 한다는 사용자 요청.
- 선택한 기술/방식: React textarea의 `onKeyDown` 이벤트에서 `event.key === 'Enter' && !event.shiftKey` 조건을 검사하고, 일반 Enter일 때만 기본 동작을 막은 뒤 기존 `submit()`을 호출한다.
- 검토한 대안: 참고 글처럼 form의 `onKeyPress`에서 처리하는 방식은 React에서 권장도가 낮은 키 이벤트이므로 선택하지 않았다. textarea의 기본 Shift+Enter 동작을 직접 문자열로 조작하는 방식도 불필요해 제외했다.
- 선택 이유: 기존 submit 흐름과 상태 검증을 재사용할 수 있고, Shift+Enter는 브라우저 기본 동작에 맡겨 구현 범위와 부작용을 줄일 수 있다.
