# 68. Tool calling 구조 설명

- 상태: 완료
- 작성일: 2026-07-01

## 질의

사용자가 아직 tool calling 구조를 이해하지 못했으므로, tool calling이 무엇인지와 현재 구현된 내용이 서비스에 어떻게 적용되는지 차근히 설명해 달라고 요청했다.

## 처리 내용

기존 `ToolPolicyService`, `AgentService`, Extension Host tool executor 구조를 기준으로 tool calling 개념을 설명했다. LLM이 직접 파일을 읽거나 명령을 실행하는 것이 아니라, 필요한 작업을 JSON 형태로 요청하고, 서비스가 허용된 도구만 대신 실행해 결과를 다시 모델에게 전달하는 구조로 정리했다.

현재 구현은 읽기 전용 v1 범위이며, `read_file`, `search_files`만 허용되고 쓰기, 명령 실행, 네트워크 접근은 허용하지 않는다고 설명했다.
