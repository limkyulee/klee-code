# 26. UI 로그 확인 방법 안내

- 상태: 완료
- 작성일: 2026-06-23

## 질의

VS Code webview UI에 대한 로그를 볼 수 있는 방법 문의.

## 처리 내용

`extension/package.json`의 빌드/watch 스크립트와 `AssistantViewProvider.ts`, `vscodeBridge.ts`를 확인했다. UI는 `extension/webview-ui` React 코드가 Vite로 `dist/webview`에 번들링되어 VS Code webview 안에서 실행되는 구조다. 따라서 UI의 `console.log`, React 런타임 오류, DOM 이벤트 확인은 일반 터미널이 아니라 VS Code의 webview 개발자 도구에서 보는 방식이 적합하다고 판단했다. Extension host 로그는 별도로 VS Code Extension Development Host의 Output/Developer Tools에서 확인해야 한다.

## 변경 파일

- `program-history/26. UI 로그 확인 방법 안내.md`

## 검증

- `rg -n "console\\.|logger|log\\(|vite|dev|webview" extension -S`
- `sed -n '55,80p' extension/package.json`
- `sed -n '1,90p' extension/src/extension-host/webview/AssistantViewProvider.ts`
- `sed -n '1,60p' extension/webview-ui/src/features/chat/api/vscodeBridge.ts`

## 결과

UI 로그 확인은 VS Code webview 개발자 도구를 중심으로 보고, extension host 로그와 백엔드 로그는 각각 별도 위치에서 확인하도록 안내했다.
