# 64. Slash 명령 레지스트리 정리

- 상태: 완료
- 작성일: 2026-06-30

## 질의

`skills` 하위 프롬프트 스킬도 `/clear`처럼 extension 단의 정해진 스킬로 넘기는 것이 맞는지, slash command를 한 곳에 모아 관리하고 싶다는 질문이었다.

## 처리 내용

`/clear` 같은 extension-local command와 `skills/*.md` 기반 prompt skill은 실행 책임이 다르다고 판단했다. `/clear`는 모델 호출 전에 extension host가 처리해야 하는 로컬 명령이고, 내부/커스텀 스킬은 백엔드가 프롬프트에 합성해야 하는 prompt skill로 유지했다.

대신 `extension/src/extension-host/chat/slashCommand.ts`에 `LOCAL_SLASH_COMMANDS` 레지스트리를 추가해 extension-local slash command를 한 곳에서 관리하도록 정리했다. `parseSlashCommand()`는 local command, prompt skill, 일반 텍스트를 구분해 반환하고, `ChatWebviewMessageHandler`는 local command를 별도 핸들러로 처리한 뒤 나머지만 백엔드 요청으로 보낸다.

`docs/DESIGN_V3.md`에도 extension-local command와 prompt skill command의 관리 위치와 실행 경로를 문서화했다.

## 기능 추가/변경 결정 기록

- 요청 배경: slash command가 늘어날 때 `/clear`와 프롬프트 스킬의 관리 위치가 흩어지지 않도록 정리하려는 요청이었다.
- 선택한 기술/방식: extension-local command는 `LOCAL_SLASH_COMMANDS` 레지스트리에서 관리하고, local command가 아닌 `/스킬명`만 백엔드 prompt skill로 전달하도록 했다.
- 검토한 대안: 모든 `skills/*.md`를 extension-local command처럼 처리하는 방식은 프롬프트 합성 책임이 백엔드에 있다는 v3 설계와 맞지 않아 제외했다.
- 선택 이유: 로컬 상태 변경 명령과 프롬프트 스킬을 분리하면 `/clear`처럼 모델 호출이 없어야 하는 명령을 안전하게 처리하면서도 slash command 정의는 한 곳에서 관리할 수 있다.
