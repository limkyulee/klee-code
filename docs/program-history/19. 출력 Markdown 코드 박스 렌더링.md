# 19. 출력 Markdown 코드 박스 렌더링

- 상태: 완료
- 작성일: 2026-06-23

## 질의

사용자가 채팅 출력 UI에서 Markdown 형식으로 전달되는 응답 중 fenced code block이 코드 박스로 보이도록 수정해 달라고 요청했다.

## 처리 내용

`MessageBubble.tsx`와 `global.css`를 확인해 응답 텍스트가 JSX 텍스트 노드로 그대로 출력되고 있음을 확인했다. 외부 Markdown 라이브러리를 추가하지 않고 webview UI 내부에 `MarkdownMessage` 컴포넌트를 추가해 코드 펜스, 제목, 목록, 문단, 인라인 코드, 굵은 텍스트를 기본 렌더링하도록 했다. 닫히지 않은 코드 펜스도 스트리밍 중 코드 박스로 보이도록 처리했다.

## 기능 추가/변경 결정 기록

- 요청 배경: LLM 응답이 Markdown으로 내려올 때 코드 예제가 일반 텍스트와 백틱으로 보여 가독성이 떨어지는 문제가 있었다.
- 선택한 기술/방식: `MarkdownMessage.tsx`에서 간단한 Markdown 블록 파서를 구현하고, fenced code block을 `figure > pre > code` 구조와 VS Code 테마 변수를 사용한 스타일로 표시했다.
- 검토한 대안: `react-markdown` 같은 외부 라이브러리 추가를 검토할 수 있었지만, 현재 의존성에 Markdown 렌더러가 없고 요구 범위가 코드 박스 표시 중심이라 의존성을 늘리지 않았다.
- 선택 이유: 변경 범위를 webview UI로 제한하면서 스트리밍 응답에도 즉시 적용할 수 있고, 기존 React/Vite 구조와 CSS 테마 변수 사용 방식에 맞다.

## 검증

- `npm run check-types` 성공
- `npm run lint` 성공
- `npm run build:webview`는 현재 셸의 Node.js가 `20.12.2`라 Vite 요구 버전(`20.19+` 또는 `22.12+`)에 미달해 실패했다. `/opt/homebrew/bin/node`의 22.x도 `libsimdjson.31.dylib` 누락으로 실행되지 않았다.

## 결과

채팅 응답의 Markdown fenced code block이 언어 라벨이 있는 코드 박스로 렌더링되도록 수정했다.
