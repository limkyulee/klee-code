# 62. Klee Code 개발 블로그 원고 작성

- 상태: 완료
- 작성일: 2026-06-30

## 질의

사용자는 `program-history`와 기존 개발 맥락을 기반으로 Klee Code의 개발 이야기, 초기 설계부터 현재 설계까지의 변화, 개발 중 겪은 문제와 해결 과정을 포트폴리오용 기술 블로그 Markdown 원고로 작성해 달라고 요청했다. 말투는 담백하고 포멀하게, AI가 쓴 것처럼 보이지 않게 작성해 달라고 했다.

## 처리 내용

- `program-history`의 1번부터 61번까지 작업 기록을 확인해 주요 흐름을 추렸다.
- `README.md`, `docs/DESIGN.md`, `docs/DESIGN_V2.md`, `docs/STRUCTURE.md`, `docs/STRUCTURE_V2.md`, `docs/TECH_STACK.md`, `extension/STRUCTURE.md`를 확인해 초기 설계와 현재 구조를 대조했다.
- 현재 `LLMGateway`, `LlmProperties`, `ModelController`, `application.yml`을 확인해 중앙 LLM Gateway와 실제 Ollama 모델 목록 조회 흐름을 현재 구현 기준으로 맞췄다.
- 개발 이야기를 기능/설계 변화 기준으로 6개 본문과 1개 목차로 나누어 `docs/development-story/`에 작성했다.
- 단순 작업 나열이 아니라 문제, 판단, 설계 변경, 결과가 이어지도록 글 흐름을 구성했다.

## 변경 파일

- `docs/development-story/00. Klee Code 개발 이야기 목차.md`
- `docs/development-story/01. 개인 AI 클라이언트가 아니라 온프레미스 AI Coding Assistant로 설계한 이유.md`
- `docs/development-story/02. VS Code Extension에서 스트리밍 UI를 다듬은 과정.md`
- `docs/development-story/03. 컨텍스트, 대화 메모리, 감사 로그를 분리한 이유.md`
- `docs/development-story/04. 로그인, 세션, 대화 히스토리를 다시 설계한 과정.md`
- `docs/development-story/05. 중앙 LLM Gateway로 제품의 성격을 다시 잡은 과정.md`
- `docs/development-story/06. 실행 환경과 배포 문제를 정리하며 남긴 기준.md`
- `program-history/62. Klee Code 개발 블로그 원고 작성.md`

## 검증

- `find docs/development-story -maxdepth 1 -type f -name '*.md' | sort -V`로 생성 파일 목록을 확인했다.
- `wc -l docs/development-story/*.md`로 문서 분량을 확인했다.
- `sed -n '1,160p' 'docs/development-story/00. Klee Code 개발 이야기 목차.md'`로 목차와 링크 형식을 확인했다.
- 문서 작성 작업이므로 빌드와 테스트는 실행하지 않았다.

## 결과

Klee Code의 개발 과정을 포트폴리오 기술 블로그로 옮겨 적기 쉬운 Markdown 원고 7개로 작성했다. 온프레미스 설계, VS Code 스트리밍 UI, 컨텍스트/감사 로그 분리, 로그인/세션/히스토리, 중앙 LLM Gateway, 실행 환경 정리 흐름으로 스토리가 이어지도록 구성했다.
