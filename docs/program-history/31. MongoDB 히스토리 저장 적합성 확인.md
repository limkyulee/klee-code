# 31. MongoDB 히스토리 저장 적합성 확인

- 상태: 완료
- 작성일: 2026-06-24

## 질의

사용자별 로그인, 모델 서버 주소 설정, 세션, 대화 히스토리 목록/상세/이어가기 기능을 저장하기에 MongoDB 사용이 적절한지 확인 요청이 있었다.

## 처리 내용

- `backend/build.gradle`에서 Spring Boot MongoDB starter와 Spring AI MongoDB chat memory starter를 확인했다.
- `backend/src/main/resources/application.yml`에서 MongoDB URI와 자동 인덱스 설정을 확인했다.
- `DESIGN.md`, `backend/STRUCTURE.md`, `ChatConfig`, `ChatService`를 확인해 현재 대화 메모리와 감사 로그가 이미 MongoDB 기반으로 동작하도록 설계된 점을 확인했다.
- `AuditLog` 문서를 확인해 현재 요청/응답 저장 구조가 `conversationId`, 질문, 답변, 컨텍스트 중심이며 사용자 소유권 정보는 아직 없다는 점을 확인했다.
- 결론적으로 MongoDB는 대화 히스토리 저장에는 적절하지만, 로그인/세션/사용자별 모델 서버 주소 저장에는 별도 도메인과 보안 설계가 필요하다고 판단했다.

## 기능 추가/변경 결정 기록

- 요청 배경: 사용자별 대화 기능을 확장하기 전에 현재 선택된 MongoDB가 저장소로 적절한지 검토해야 했다.
- 선택한 기술/방식: MongoDB를 유지하되 사용자, 세션, 모델 서버 설정, 대화방, 메시지 컬렉션을 분리하는 방향을 권장했다.
- 검토한 대안: 관계형 DB를 도입하는 대안도 가능하지만, 현재 프로젝트가 이미 MongoDB와 Spring AI MongoDB ChatMemory를 사용하고 있어 초기 구현 범위가 커진다.
- 선택 이유: 대화 메시지와 코드 컨텍스트는 스키마가 변하기 쉽고 문서형 저장에 맞으며, 기존 MongoDB 인프라와 자동 ChatMemory 구성을 재사용할 수 있다. 단, 인증 정보와 세션은 인덱스, TTL, 비밀번호 해시, 사용자 소유권 검증이 필수다.

## 결과

MongoDB는 사용자별 대화 히스토리 목록/상세/이어가기 저장소로 적절하나, 로그인과 세션까지 포함하려면 사용자 소유권, 세션 만료, 비밀번호 해시, 인덱스 설계를 추가해야 한다고 정리했다.
