## 질의

`DESIGN.md`를 기준으로 현재 프로젝트가 어느 정도까지 진행됐는지 확인해 달라는 요청.

## 처리 내용

- `DESIGN.md`와 `README.md`의 Phase 정의를 먼저 확인했다.
- 백엔드의 `ChatConfig.java`, `ChatService.java`, `ChatController.java`를 읽어 ChatClient 추상화, ChatMemory 연동, `/chat` 및 `/chat/stream` 구현 여부를 확인했다.
- 확장 쪽 `package.json`, `webviewMessageHandler.ts`, `App.tsx`를 확인해 커맨드 등록, `conversationId` 생성, SSE 스트리밍 UI가 동작하는 구조인지 대조했다.
- `application.yml`과 `docker-compose.yml`을 확인해 모델 토글과 현재 패키징 범위를 점검했다.
- `chat/repository/package-info.java`를 확인해 감사 로그와 컨텍스트 조회가 아직 Phase 3 이후로 남아 있음을 확인했다.
