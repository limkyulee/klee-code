# 70. Tool calling backend 역할 설명

- 상태: 완료
- 작성일: 2026-07-01

## 질의

프롬프트에 tool 설명을 붙이고 모델 응답에서 `<klee_tool_call>` 태그를 찾는 처리를 원래 백엔드에서 하는 것이 맞는지 질문했다.

## 처리 내용

일반적으로 tool calling의 orchestration, 정책 검사, tool 결과 대기, 모델 재호출은 백엔드가 담당하는 것이 자연스럽다고 설명했다. 다만 Klee Code처럼 VS Code 확장 기반 서비스에서는 실제 로컬 워크스페이스 파일 접근은 백엔드가 아니라 Extension Host가 실행하는 것이 권한 경계상 적합하다고 정리했다.

현재 구현은 백엔드가 tool 호출을 관리하고 Extension Host가 로컬 파일 읽기/검색을 실행하는 분리 구조라고 설명했다. 또한 현재의 `<klee_tool_call>` 태그 방식은 native function/tool calling API가 아니라 프롬프트 기반의 단순 v1 구현이라는 점을 구분했다.

## 검증

테스트는 실행하지 않았다. 설명 요청이므로 코드 변경 없이 구조를 설명했다.

## 결과

백엔드는 tool calling의 관리자 역할을 하고, Extension Host는 로컬 파일 접근 실행자 역할을 하는 것이 현재 서비스 구조에 맞다고 답변했다.
