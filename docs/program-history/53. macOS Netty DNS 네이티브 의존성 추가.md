# 53. macOS Netty DNS 네이티브 의존성 추가

- 상태: 완료
- 작성일: 2026-06-29

## 질의

백엔드 로그에 `io.netty.resolver.dns.macos.MacOSDnsServerAddressStreamProvider` 네이티브 라이브러리를 로드하지 못해 macOS DNS 해석이 부정확할 수 있다는 오류가 출력되는 문제를 해결해 달라는 요청이었다.

## 처리 내용

`backend/build.gradle`의 Spring Boot, Spring AI, Netty 관련 의존성 구성을 확인했다. `spring-ai-starter-model-ollama`가 WebClient/Reactor Netty를 통해 Netty DNS resolver를 사용하는 구조로 확인되어, macOS 네이티브 DNS resolver 런타임 의존성을 추가했다.

## 기능 추가/변경 결정 기록

- 요청 배경: macOS에서 Netty가 네이티브 DNS provider를 찾지 못해 오류 로그를 출력했다.
- 선택한 기술/방식: Gradle `runtimeOnly`로 `io.netty:netty-resolver-dns-native-macos`의 `osx-aarch_64`, `osx-x86_64` classifier를 추가했다.
- 검토한 대안: 특정 CPU 아키텍처 하나만 추가하는 방식을 고려했지만, 개발 환경이 Apple Silicon/Intel Mac 중 어느 쪽이어도 동작하도록 두 classifier를 함께 추가했다.
- 선택 이유: Spring Boot의 dependency management가 Netty 버전을 관리하므로 버전 직접 지정 없이 기존 의존성 관리 방식과 일관성을 유지할 수 있고, 런타임 의존성만 추가해 코드 변경 범위를 최소화했다.

## 변경 파일

- `backend/build.gradle`
- `program-history/53. macOS Netty DNS 네이티브 의존성 추가.md`

## 검증

`./gradlew dependencyInsight --dependency netty-resolver-dns-native-macos --configuration runtimeClasspath`를 실행해 `io.netty:netty-resolver-dns-native-macos:4.2.15.Final`이 runtimeClasspath에 포함되는 것을 확인했다.

`./gradlew test`를 실행했고 `BUILD SUCCESSFUL`로 완료됐다.

## 결과

macOS에서 Netty DNS resolver 네이티브 라이브러리가 런타임 클래스패스에 포함되도록 수정했고, Gradle 의존성 해석과 테스트를 통과했다.
