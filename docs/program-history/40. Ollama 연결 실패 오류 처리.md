# 40. Ollama 연결 실패 오류 처리

- 상태: 완료
- 작성일: 2026-06-28

## 질의

첨부 로그의 `Connection refused: localhost/127.0.0.1:11434` 오류를 확인하고 조치해 달라는 요청이었다.

## 처리 내용

`ChatService`, `ChatController`, 사용자별 Ollama `ChatClient` 생성 경로, 공통 API 예외 처리, 확장 스트림 파서를 확인했다.

Ollama 서버가 꺼져 있거나 모델 서버 URL이 잘못된 경우 내부 `WebClientRequestException`이 그대로 노출되던 문제를 해결했다. 모델 연결 실패를 `MODEL_SERVER_UNAVAILABLE` 503 API 오류로 변환하고, 스트리밍 응답에서는 SSE `error` 이벤트로 전달한 뒤 정상 종료하도록 변경했다. 확장 스트림 파서는 해당 `error` 이벤트를 예외로 변환해 UI의 기존 오류 표시 흐름을 타게 했다.

## 기능 추가/변경 결정 기록

- 요청 배경: Ollama 기본 포트 `11434` 연결 거부 스택 트레이스가 발생해 사용자에게 원인과 조치가 명확히 전달되지 않았다.
- 선택한 기술/방식: `ChatModelExceptionMapper` 컴포넌트를 추가해 `WebClientRequestException`/`ConnectException` 계열을 `ApiException(HttpStatus.SERVICE_UNAVAILABLE, MODEL_SERVER_UNAVAILABLE)`으로 매핑했다. `/chat/stream`은 SSE `error` 이벤트를 보내고, 확장 클라이언트는 이를 `Error`로 throw한다.
- 검토한 대안: Ollama 실행 여부를 사전 health check로 검사하는 방식도 가능하지만, 실제 호출 시점 오류와 중복되고 요청별 모델 설정 URL에 대한 실패 처리가 여전히 필요해 선택하지 않았다.
- 선택 이유: 기존 `ApiException`/`ApiExceptionHandler` 구조와 맞고, 일반 요청과 스트리밍 요청 모두 같은 메시지를 사용할 수 있으며 변경 범위가 작다.

## 검증

- `cd backend && ./gradlew test`: 성공
- `cd extension && npm test -- --runInBand`: 실패. 테스트 컴파일, 타입체크, 린트는 통과했지만 빌드 단계에서 현재 Node.js `20.12.2`가 Vite 요구 버전 `20.19+` 및 프로젝트 `engines.node >=22.12.0`보다 낮아 중단됐다.

## 결과

Ollama 연결 실패가 내부 스택 트레이스 대신 “Model server is unreachable at ... Start Ollama or update the model server URL.” 메시지로 전달되도록 수정했다.
