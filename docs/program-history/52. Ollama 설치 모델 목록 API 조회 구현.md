# 52. Ollama 설치 모델 목록 API 조회 구현

- 상태: 완료
- 작성일: 2026-06-29

## 질의

`application.yml`에 정의한 모델 목록 대신 실제 Ollama 서버에 설치된 LLM 목록을 조회하도록 모델 목록 API를 수정해달라고 요청했다.

## 처리 내용

`LLMGateway.java`, `LlmProperties.java`, `application.yml`, 테스트 설정과 관련 테스트를 확인했다. Spring AI의 `OllamaApi.listModels()`가 Ollama `GET /api/tags`를 호출하는 것을 확인하고, `OllamaApi`를 Spring 빈으로 분리했다. `/models` 응답과 모델 허용 여부 검증이 YAML의 `klee.llm.models`가 아니라 Ollama 설치 모델 목록을 기준으로 동작하도록 변경했다. Ollama 연결 실패 시 503 `MODEL_SERVER_UNAVAILABLE` 응답으로 매핑되도록 처리했다.

## 기능 추가/변경 결정 기록

- 요청 배경: UI/백엔드에서 설정 파일의 정적 모델 목록이 아니라 실제 Ollama에 설치된 모델 목록을 보여줘야 했다.
- 선택한 기술/방식: Spring AI `OllamaApi.listModels()`를 사용해 Ollama `/api/tags` 결과를 조회하고, 기존 `/models` API 응답 형식인 `ModelResponse`로 변환했다.
- 검토한 대안: MCP를 통한 조회를 검토했지만, 이 요구사항은 LLM 도구 호출이 아니라 백엔드의 서버 상태 조회이므로 Ollama REST API 직접 호출이 더 적합하다고 판단했다.
- 선택 이유: 기존 Spring AI Ollama 의존성을 재사용하므로 구현 범위가 작고, `/models` API 계약을 유지하면서 실제 설치 상태를 반영할 수 있다.
