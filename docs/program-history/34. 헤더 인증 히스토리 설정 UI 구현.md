# 34. 헤더 인증 히스토리 설정 UI 구현

- 상태: 완료
- 작성일: 2026-06-24

## 질의

초기 로그인 화면과 coder 설정 화면을 먼저 보이지 않게 하고, 헤더 우측 아이콘으로 대화 히스토리, 설정, 새 대화 기능을 제공하도록 요청했다. 히스토리 팝업은 계정별 대화 목록을 반응형으로 보여주고, 설정 팝업은 로그인/회원가입/로그아웃과 Coder URL 및 모델 설정을 처리해야 한다.

## 처리 내용

`ChatView`, 웹뷰 프로토콜, 확장 호스트 메시지 핸들러, 백엔드 감사 로그 도메인을 확인했다. 웹뷰가 인증 또는 모델 설정 상태에 따라 전체 화면을 전환하던 흐름을 제거하고 항상 채팅 셸을 먼저 렌더링하도록 변경했다. 헤더 우측에 히스토리, 설정, 새 대화 아이콘을 추가했고, 히스토리 팝업과 설정 팝업을 반응형 폭으로 구현했다.

계정별 히스토리 조회를 위해 감사 로그 기반의 `/audit/chat-history` API를 추가하고, 확장 호스트에서 인증 토큰으로 조회한 뒤 웹뷰에 `CHAT_HISTORY` 메시지로 전달하도록 연결했다. 설정 팝업에는 로그인 상태 표시, 로그아웃, Coder URL 및 모델 저장 폼을 배치했고, 로그아웃 또는 비로그인 상태에서는 로그인/회원가입 폼을 표시하도록 했다.

## 기능 추가/변경 결정 기록

- 요청 배경: 사용자가 초기 진입 시 로그인/설정 화면을 먼저 보지 않고, 참고 이미지처럼 헤더 아이콘 기반으로 인증과 히스토리 기능을 사용하길 원했다.
- 선택한 기술/방식: 기존 React 웹뷰와 VS Code `postMessage` 프로토콜을 확장하고, 기존 감사 로그 MongoDB 문서에서 계정별 최근 대화 목록을 조회하는 API를 추가했다.
- 검토한 대안: 프론트엔드 로컬 상태만으로 히스토리를 표시하는 방식은 재시작 후 계정별 기록을 복원할 수 없어 제외했다. 별도 히스토리 컬렉션을 새로 만드는 방식은 현재 감사 로그가 이미 질문, 상태, 시각, 사용자 ID를 보유하고 있어 범위 대비 중복이 커 제외했다.
- 선택 이유: 기존 인증 세션, 모델 설정 API, 감사 로그 저장 구조와 일관되며, 새 저장소를 만들지 않아 구현 범위가 작고 유지보수성이 높다.
