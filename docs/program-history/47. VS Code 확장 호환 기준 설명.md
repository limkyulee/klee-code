# 47. VS Code 확장 호환 기준 설명

- 상태: 완료
- 작성일: 2026-06-29

## 질의

VS Code 확장을 최신 버전에 맞추는 것이 맞는지, 현재 시스템 버전에 맞춰 호환 버전을 낮추는 것이 맞는지 판단 기준을 질문했다.

## 처리 내용

현재 문제는 사용 중인 VS Code 1.118.1보다 확장 매니페스트의 `engines.vscode` 요구 버전 `^1.125.0`이 높아 개발용 확장 로드가 차단된 상황이라고 설명했다. 확장이 1.125 전용 API를 사용하지 않는다면 최소 지원 버전을 1.118로 낮추는 것이 개발 환경과 배포 호환성 측면에서 자연스럽고, 최신 VS Code API가 필요해질 때는 VS Code와 타입 패키지를 함께 올리는 것이 맞다고 정리했다.

## 결과

확장 호환 버전은 최신 버전 고정이 아니라 실제 사용하는 VS Code API와 지원 대상 사용자 버전을 기준으로 정해야 한다고 안내했다.
