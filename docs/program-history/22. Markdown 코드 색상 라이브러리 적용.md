# 22. Markdown 코드 색상 라이브러리 적용

- 상태: 완료
- 작성일: 2026-06-23

## 질의

`MarkdownMessage.tsx`의 코드 스타일 색상 처리를 직접 하드코딩하지 말고, 기존 UI 라이브러리나 Markdown 코드 스타일 구현을 가져다 쓰도록 개선해 달라는 요청.

## 처리 내용

`extension/webview-ui/src/features/chat/components/MarkdownMessage.tsx`를 확인해 정규식 기반 커스텀 코드 토크나이저와 `syntax-*` span 렌더링이 직접 구현되어 있음을 확인했다. 프로젝트에는 기존 Markdown/코드 하이라이트 의존성이 없어 `highlight.js`를 추가하고, 코드 블록 렌더링을 `highlight.js/lib/common` 기반으로 변경했다.

`extension/webview-ui/src/main.tsx`에서 `highlight.js/styles/github-dark.css` 테마를 import하도록 연결했다. `extension/webview-ui/src/styles/global.css`에서는 직접 색상을 지정하던 `syntax-*` 규칙을 제거하고, 코드 블록 레이아웃은 유지하되 highlight.js 테마가 토큰 색상을 담당하도록 정리했다.

## 기능 추가/변경 결정 기록

- 요청 배경: 기존 코드 하이라이트 색상이 컴포넌트/CSS에 직접 하드코딩되어 품질과 유지보수성이 낮다는 지적이 있었다.
- 선택한 기술/방식: `highlight.js` 의존성을 추가하고, `hljs.highlight`와 `hljs.highlightAuto`로 코드 HTML을 생성한 뒤 라이브러리 CSS 테마를 적용했다.
- 검토한 대안: 현재 정규식 토크나이저를 유지하면서 CSS 변수만 조정하는 방식은 하드코딩 구조를 계속 유지하므로 선택하지 않았다. 더 큰 Markdown 렌더러 교체는 요청 범위보다 변경 폭이 커서 제외했다.
- 선택 이유: 코드 하이라이트 책임을 검증된 라이브러리로 넘기면서 기존 Markdown 파서, 코드 복사 UI, 레이아웃 구조는 유지할 수 있어 변경 범위가 작고 안정적이다.
