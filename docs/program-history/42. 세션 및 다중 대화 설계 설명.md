# 42. 세션 및 다중 대화 설계 설명

- 상태: 완료
- 작성일: 2026-06-28

## 질의

백엔드 개발 경험이 1년차인 주니어 개발자도 이해할 수 있도록 이 서비스의 세션 및 다중 대화 설계안과 기술을 다시 설명해 달라고 요청했다.

## 처리 내용

기존 채팅 구조에서 로그인 세션, 대화 세션, 스트리밍 요청 세션을 구분해 설명하고, LLM 서버가 하나일 때 다중 대화가 어떤 의미인지 정리했다. 앱 내부의 대화별 상태 분리와 LLM 서버의 실제 병렬 처리 능력이 별개라는 점을 중심으로 설계안을 설명했다.
