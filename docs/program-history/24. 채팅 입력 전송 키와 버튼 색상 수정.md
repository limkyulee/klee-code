# 24. 채팅 입력 전송 키와 버튼 색상 수정

- 상태: 완료
- 작성일: 2026-06-23

## 질의

사용자가 `ChatInput.tsx`에서 입력값이 있으면 전송 아이콘이 자동 활성화되고, 활성 색상은 파란색이 아닌 흰색으로 보이게 하며, Enter는 채팅 전송, 실제 줄바꿈은 Shift+Enter로 동작하게 해 달라고 요청했다.

## 처리 내용

- `extension/webview-ui/src/features/chat/components/ChatInput.tsx`의 textarea 키 입력 처리를 확인했다.
- 기존 `Ctrl/Cmd+Enter` 전송 조건을 일반 `Enter` 전송으로 변경하고, `Shift+Enter`는 기본 textarea 줄바꿈 동작을 유지하게 했다.
- 한글 등 IME 조합 중 Enter가 전송으로 처리되지 않도록 `event.nativeEvent.isComposing` 검사를 추가했다.
- `extension/webview-ui/src/styles/global.css`에서 활성 전송 버튼 배경을 흰색으로 변경하고, 빈 입력으로 비활성화된 버튼은 투명 배경과 설명 색상으로 보이게 조정했다.

## 기능 추가/변경 결정 기록

- 요청 배경: 채팅 입력 UX를 일반 메신저처럼 Enter 전송, Shift+Enter 줄바꿈 방식으로 바꾸고 전송 아이콘 활성 색상을 흰색으로 맞추기 위한 요청이었다.
- 선택한 기술/방식: React textarea `onKeyDown`에서 Enter 입력을 가로채 `submit()`을 호출하고, CSS의 `.send` 활성/비활성 상태 스타일을 분리했다.
- 검토한 대안: form submit만 유지하는 방식은 textarea에서 Enter 줄바꿈과 충돌해 제외했다. VS Code 기본 버튼 색상 변수를 유지하는 방식은 테마에 따라 파란색으로 보일 수 있어 제외했다.
- 선택 이유: 기존 상태(`text`, `pending`)와 submit 흐름을 유지하면서 변경 범위를 작게 가져갈 수 있고, IME 조합 예외 처리로 한국어 입력 중 오전송 가능성을 줄였다.
