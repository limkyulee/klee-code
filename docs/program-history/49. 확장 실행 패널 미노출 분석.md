# 49. 확장 실행 패널 미노출 분석

- 상태: 완료
- 작성일: 2026-06-29

## 질의

VS Code Extension Development Host 실행 로그에 deprecation warning, Bun socket 충돌, unhandled promise 오류가 보이고, Run Extension 후 Klee Code 패널이 보이지 않는 문제를 분석해 달라는 요청.

## 처리 내용

`extension/package.json`, `extension/src/extension-host/extension.ts`, `AssistantViewProvider`, Vite 설정, VS Code launch/task 설정을 확인했다.

`npm run compile`을 실행해 Klee Code 확장 자체의 타입 체크, 린트, extension/webview 빌드가 성공함을 확인했다. 로그의 `oven.bun-vscode` 경로와 Claude/Mermaid 출력은 Klee Code 확장 빌드 오류가 아니라 개발 호스트에 함께 로드된 다른 VS Code 확장 로그로 판단했다.

개발 실행 시 외부 확장 오류가 섞이지 않도록 `--disable-extensions`를 추가하고, 실행 전 `npm run compile`이 수행되도록 launch/task 설정을 보강했다. 저장소 루트를 열고 실행하는 경우도 지원하도록 루트 `.vscode` launch/task 설정을 추가했다.

## 기능 추가/변경 결정 기록

- 요청 배경: Run Extension 후 Klee Code 패널이 보이지 않고, 외부 확장 오류 로그가 함께 출력되어 원인 분리가 어려웠다.
- 선택한 기술/방식: Extension Development Host 실행 인자에 `--disable-extensions`를 추가하고, `preLaunchTask`로 `npm run compile`을 연결했다. 루트 워크스페이스에서도 `extension` 하위 확장을 실행할 수 있는 launch 설정을 추가했다.
- 검토한 대안: `activationEvents`를 다시 명시하는 방법도 가능하지만, 현재 VS Code 버전에서는 view/command contribution 기반 자동 activation이 동작하므로 우선 실행 환경 격리와 빌드 보장을 선택했다.
- 선택 이유: 외부 확장 런타임 오류와 Klee Code 로드 문제를 분리할 수 있고, 오래된 `dist` 또는 누락된 빌드 산출물로 인한 개발 실행 실패를 줄이는 가장 작은 변경이다.
