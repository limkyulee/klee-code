# 16. 스트리밍 토큰 공백 보존 전송 수정

- 상태: 완료
- 작성일: 2026-06-23

## 질의

스트리밍 응답 chunk 로그에는 앞 공백이 포함되어 있는데 화면에는 단어가 붙어서 표시되는 문제를 수정해 달라는 요청이었다.

## 처리 내용

`ChatController.java`, `chatApiClient.ts`, `messageReducer.ts`, `MessageBubble.tsx`, `global.css`를 확인했다. 백엔드 로그의 token 앞 공백은 살아 있었고, SSE `data:` 필드의 구분자 공백 처리와 raw 문자열 전송 방식이 충돌할 수 있다고 판단했다. `ChatController`에서 SSE data를 그대로 보내지 않고 Jackson `ObjectMapper`로 JSON 문자열 인코딩해 보내도록 변경했다. 클라이언트는 이미 JSON 문자열을 `JSON.parse`로 복원하므로 앞 공백이 포함된 token이 그대로 누적된다.

## 기능 추가/변경 결정 기록

- 요청 배경: 모델 응답 chunk의 앞 공백이 화면 렌더링 전에 사라져 `Hello!I'macoding...`처럼 붙어서 표시됐다.
- 선택한 기술/방식: SSE payload를 JSON 문자열로 인코딩해 전송하고, 기존 클라이언트의 JSON parse 경로로 복원한다.
- 검토한 대안: 클라이언트에서 `trimStart()`만 제거하는 방식은 이미 일부 반영되어 있었지만, SSE raw data의 첫 공백 구분자 규칙과 계속 충돌할 수 있어 충분하지 않았다.
- 선택 이유: 백엔드 전송 포맷에서 공백을 명확히 보존하므로 UI 렌더링이나 chunk 누적 로직에 추가 보정이 필요 없고, 기존 클라이언트 파서와도 호환된다.
