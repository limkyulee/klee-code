# 14. Codex 히스토리 경로 오류 수정

- 상태: 완료
- 작성일: 2026-06-23

## 질의

사용자는 Codex가 `.codex/` 폴더 안에 이미 있는데 왜 읽지 못했는지 지적했다. 이전 응답에서 루트 `AGENTS.md`와 루트 `program-history`가 없다고 판단한 것이 잘못이었다.

## 처리 내용

- `.codex/AGENTS.md`가 존재하고, 히스토리 위치가 `.codex/program-history`로 명시되어 있음을 확인했다.
- `.codex/program-history`에 기존 1번부터 12번까지의 기록이 이미 존재함을 확인했다.
- 내가 잘못 생성한 루트 `AGENTS.md`와 루트 `program-history` 내부 기록 파일을 제거했다.
- 누락됐던 직전 `extension/` 구조 정리 작업 기록을 `.codex/program-history/13. extension 폴더 구조 및 역할 명명 정리.md`로 작성했다.
- 현재 경로 오류 수정 작업을 이 파일로 작성했다.

## 기능 추가/변경 결정 기록

- 요청 배경: 사용자가 `.codex` 안에 이미 지침과 히스토리 폴더가 있는데 왜 Codex가 이를 읽지 않았는지 지적했다.
- 선택한 기술/방식: 기존 `.codex/AGENTS.md` 지침을 기준으로 올바른 히스토리 경로인 `.codex/program-history`에 기록을 이어 쓰고, 잘못 만든 루트 중복 파일을 제거했다.
- 검토한 대안: 루트 `AGENTS.md`와 루트 `program-history`를 그대로 두는 방법도 있었지만, `.codex/AGENTS.md`의 기존 지침과 충돌하고 기록 위치가 이중화된다.
- 선택 이유: 이미 저장소의 Codex 지침은 `.codex/AGENTS.md`에 정리되어 있으므로, 해당 단일 기준을 유지하는 편이 혼선을 줄이고 기존 1~12번 기록과 연속성을 보장한다.

## 변경 파일

- `.codex/program-history/13. extension 폴더 구조 및 역할 명명 정리.md`
- `.codex/program-history/14. Codex 히스토리 경로 오류 수정.md`
- `AGENTS.md`
- `program-history/1. extension 폴더 구조 및 역할 명명 정리.md`
- `program-history/2. Codex 질의 히스토리 생성 누락 수정.md`

## 검증

- `find .codex -maxdepth 4 -type f | sort`로 `.codex/AGENTS.md`와 기존 히스토리 파일을 확인했다.
- `sed -n '1,220p' .codex/AGENTS.md`로 실제 지침 내용을 확인했다.

문서와 기록 경로 수정 작업이므로 빌드나 테스트는 실행하지 않았다.

## 결과

히스토리 기록 기준을 기존 `.codex/AGENTS.md`와 `.codex/program-history`로 바로잡았다. 잘못 만든 루트 중복 파일은 제거했고, 누락된 작업 기록을 13번과 14번으로 올바른 위치에 추가했다.
