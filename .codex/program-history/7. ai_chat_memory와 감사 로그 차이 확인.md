# 7. ai_chat_memory와 감사 로그 차이 확인

- 상태: 완료
- 작성일: 2026-06-23

## 질의

지금 `ai_chat_memory`에 기록되는 것과 Phase 3의 감사 로그가 같은 것인지, 다른 것인지 확인 요청이 있었다.

## 처리 내용

- `DESIGN.md`와 `README.md`에서 ChatMemory와 감사 로그의 역할을 확인했다.
- `backend/src/main/java/com/kleecode/backend/config/ChatConfig.java`를 확인해 `MessageChatMemoryAdvisor` + `ChatMemory`가 대화 연속성용으로만 사용되는 것을 확인했다.
- `backend/src/main/java/com/kleecode/backend/chat/dto/ChatRequest.java`를 확인해 `conversationId`가 메모리 조회 키임을 확인했다.
- `backend/src/main/java/com/kleecode/backend/chat/audit/AuditLog.java`와 `AuditLogService.java`를 확인해 감사 로그가 별도 `audit_logs` 컬렉션에 저장되며, 요청 시작/성공/실패와 컨텍스트, 외부전송 여부를 기록하는 구조임을 확인했다.
- 결론적으로 `ai_chat_memory`와 감사 로그는 목적과 컬렉션이 다른 별도 저장소라고 정리했다.

## 기능 추가/변경 결정 기록

- 요청 배경: `ai_chat_memory`에 들어가는 데이터와 Phase 3 감사 로그가 같은지 확인이 필요했다.
- 선택한 기술/방식: 코드에 있는 `ChatMemory`/`audit_logs` 구현을 기준으로 역할 분리를 설명했다.
- 검토한 대안: 두 저장소를 같은 기록으로 볼 수도 있었지만, 실제 코드와 설계 문서가 대화 메모리와 감사 로그를 분리하고 있어 해당 해석은 맞지 않았다.
- 선택 이유: `ai_chat_memory`는 모델이 다시 읽는 대화 메모리이고, 감사 로그는 사람이 감사/추적하기 위한 기록이라 목적이 다르다.
