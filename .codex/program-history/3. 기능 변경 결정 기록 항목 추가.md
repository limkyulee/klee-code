# 3. 기능 변경 결정 기록 항목 추가

- 상태: 완료
- 작성일: 2026-06-23

## 질의

사용자는 `AGENTS.md`에 기능을 추가하거나 변경한 경우, 어떤 요청으로 기능 추가 또는 변경을 결정했는지, 선택한 기술은 무엇인지, 함께 고민한 대안은 무엇인지, 최종 기술을 선택한 이유는 무엇인지도 히스토리에 정리하도록 요청했다.

## 처리 내용

- 기존 `AGENTS.md`의 `Program History` 기록 형식을 확인했다.
- 히스토리 템플릿에 `기능 추가/변경 결정 기록` 섹션을 추가했다.
- 해당 섹션에 요청 배경, 선택한 기술/방식, 검토한 대안, 선택 이유를 기록하도록 명시했다.

## 기능 추가/변경 결정 기록

- 요청 배경: 사용자가 단순 작업 결과뿐 아니라 기능 변경 의사결정의 맥락과 기술 선택 근거까지 추적 가능하게 남기길 원했다.
- 선택한 기술/방식: 루트 `AGENTS.md`의 히스토리 템플릿에 기능 변경 전용 섹션을 추가하는 방식을 선택했다.
- 검토한 대안: 별도 의사결정 로그 파일을 만드는 방식도 가능하지만, 히스토리와 결정 근거가 분리되어 작업 단위 추적성이 떨어진다고 판단했다.
- 선택 이유: 기존 `program-history` 기록 흐름을 유지하면서 필요한 정보만 확장할 수 있고, 매 작업 기록 안에서 요청, 변경, 검증, 결정 근거를 한 번에 확인할 수 있어 유지보수성이 좋다.

## 변경 파일

- `AGENTS.md`
- `program-history/3. 기능 변경 결정 기록 항목 추가.md`

## 검증

문서 지침 변경이므로 별도 테스트나 빌드는 실행하지 않았다. 변경된 `AGENTS.md` 내용을 확인했다.

## 결과

앞으로 기능 추가 또는 기존 기능 변경이 발생하면 `program-history` 기록에 요청 배경, 선택한 기술/방식, 검토한 대안, 선택 이유가 함께 정리되도록 지침이 갱신되었다.
